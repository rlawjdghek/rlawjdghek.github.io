하지만 Task에 따라 다르므로 그냥 구형하다가 생각 안나면 보도록 하자.

1. input을 [-1, 1]로 normalizing하고 generator의 output을 tanh를 사용 한다.

2. generator의 트레이닝시에 label과 loss function을 flip 한다.

3. generator에 대한 input으로서  사양하는 노이즈는 gasussian random number를 샘플로 한다.

4. batch norm statistics 계산을 위해서 real또는 fake끼리만 mini batch로 사용 한다.

5. generator와 discriminator에서 Leaky ReLU를 사용 한다.

6. down-sampling을 위해서 average pooling과 stride를 사용하고 up-sampling을 위해서 stride 와 ConvTranspose2D를 사용 한다.

7. discriminator에서 작은 random noise로 label smoothing을 사용 한다.

8. discriminator의 label에 random noise를 추가 한다.

9. 그러지 말아야할 이유가 있지 않은 한 DCGAN architecture를 사용한다.

10. discriminator에서 loss가 0.0인 것은 실패다.

11. generator의 loss가 일정하게 감소할 경우, 아마도 discriminator가 garbage image로 인해 잘못되고 있는 것일 수 있다.

12. label을 가지고 있다면 사용하는 것이 좋다.

13. discriminator의 input에 noise를 넣고 시간에 따라서 noise를 감쇄 시킨다.

14. training과 generation에서 50% dropout을 사용 한다.

15. CNN에서 downsampling을 위해서 max-pooling 레이어를 사용하는 것은 일반적이다. 
GAN에서는, discriminator에서 downsampling을 위해 pooling layer를 사용하지 않도록 하고 stride와 함께 convlutional layer를 사용하도록 권장한다.

16. 비슷하게 fractional stride(deconvolutional layers)은 upsampling을 위해서 generator에서 사용될 수 있다.

17. 보통은 Convolutional layer에서 feature를 추출하고 나서 추출된 feature를 위해서 모델의 output layer전에 fully-connected layer를 사용한다.
대신, GAN에서는 fully-connected layer는 사용되지 않는다. Disciriminator에서는 convolutional layer는 flattend 되고 output layer에 직접적으로 연결 된다.

18. 비슷하게 random gaussian input vector는 generator에 전달되고 multi-dimenstional tensor로 reshape 되어서 직접적으로 첫번째 convolutional layer을 통해서 
upscaling이 될 수 있는 상태가 된다.

19. Batch normalization은 이전 레이어로부터의 activation이 zero mean과 unit variance가 되도록 한다. 이러한 효과는 학습과정을 안정적으로 가져갈 수 있도록 한다.
Batch normalization은 deep convolutional neural network를 훈련시킬 때 필수품이 됬다. GAN 또한 다르지 않다. Batch norm layer는 generator의 ouput과 
discriminator의 input을 제외하고, discriminator와 generator model 모두에 사용하도록 권장된다.
그러나 Batchnrom을 모든 layer에 적용하는 것은 sample에 변동을 주며  모델을 불안정하게 한다. 그렇기 때문에 discriminator의 input과 generator output에 batch norm을 
적용하는 것은 피해야 한다.

20. ReLU와 같은 activation의 사용은 deep convolutional neural network에서 gradient vanishing problem을 해결하고자 함이다. 
그리고 sparse activation으로 촉진시키기 위함이다(많은 0 값들). 
ReLU는 generator에 권장된다. 그렇지만 discriminator에서는 아니다. 대신에, ReLU의 변형체 예를들어 Leaky ReLU와 같은 것들을 discriminator에 사용하도록 권장 한다.
ReLU는 output layer를 제외하고(tanh 사용) generator에 사용된다. [ …] discriminator에서는 leaky rectified activation이 잘 동작한다는 점을 확인 했다.

21. output layer에서 generator는 hyperbolic tangent를 사용한다. generator와 discriminator의 input은 [-1, 1]의 범위로 스케일 된다. (해야 한다)
tanh activation function의 범위 [-1, 1]으로 스케일링 하는 것을 제외하면 training images의 preprocessing은 없다.

22. model weight는 작은 guassian random value로 초기화 되었고 disriminator의 Leaky ReLU의 slope는 0.2의 값으로 초기화 됐다.
모든 weight는 0을 중심으로 표준편차 0.02의 normal distribution으로 초기화되었다. LeakyReLU에서, 0.2의 작은 slope로 모델 전체에 적용 됬다.

23. generator와 discriminator는 stochastic gradient descent를 사용해서 128의 batch size로 훈련된다.
모든 모델은 mini-batch stochastic gradient descent(SGD)에서 batch size 128로 트레이닝 된다.

24. 특히, stochastic gradient descent의l Adam 버전은 learning rate 0.0002와 momentum (beta1) 0.5로 트레이닝 된다.
우리는 hyperpameter를 변경한 Adam optimizer를 사용했다. learning rate 0.001은 너무 높고 0.0002가 적당하다는 점을 찾았고,
 momentum B1은 0.9에서 트레이닝이 변동성이 크고 불안정한 반면 0.5에서 트레이닝이 안정적이었다.
