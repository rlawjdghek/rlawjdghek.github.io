### Abstract & Introduction

StyleGAN1 (편의상 StyleGAN1이라고 하자. )이 생성한 이미지에서 고질적으로 보이는 단점은 Droplet artifact로 아래 그림을 보면 차에 이상한 덩어리 같은 부분이 있는데 (물방울 같은 덩어리)
![](/assets/images/2021-07-08-StyleGAN2/1.JPG)
이 문제를 해결하기 위해 다음 2가지 측면에서 개선책을 제시한다. 
1. Block Architecture 
2. Normalization

두번째 개선책은 이미지 향상이다. 이 부분도 아래 3가지로 개선하였다.
1. Regularization
2. Model Architecture
3. Model Scale
논문의 흐름은 위 5가지를 구체적으로 설명한 뒤, 결과를 보여주고 끝.

결과는 FID, PPL, Pricision, Recall로 보여주고, 마지막 Latent projection에서는 LPIPS를 사용한다.

### Removing Normalization artifacts
droplet을 제거해보자.
먼저 저자들은 StyleGAN1에서 normalization을 없앴더니 droplet 없이 잘 나왔다. 따라서 magnitude가 상대적인 것을 없애는 normalization 방법에 문제가 있다고 판단.
기존의 방법은 Instance Normalization에 adaptive를 더하여 normalization 효과를 더했지만 이것이 오히려 역효과를 불러왔다는 것이다.

따라서 어느정도는 generator에서 생성한 feature가 strength를 갖고가야 한다고 생각하였다.
#### Generator architecture revisited
feature의 strength를 유지하기 위해 첫번째로 고안한 방법은 블록의 구조 변화이다. 
![](/assets/images/2021-07-08-StyleGAN2/2.JPG)
우선 위의 그림을 먼저 보자.

(a): StyleGAN1의 블록 구조. PG-GAN에서 사용한 growing learning을 위하여 해상도가 증가하는 쪽으로 블록이 쌓임.

(b): StyleGAN1의 블록구조를 좀 더 자세히 표기.

(c): 새로 고안된 구조.

(d): 새로 고안된 구조를 좀 더 자세히 표기.

그림 (b)를 참고하면, 기존에는 순서가 

modulation (mod), conv, bias, norm

이었다. 가장 주목해야 할 곳은 normalization을 한 뒤, mod를 하고, bias를 한 뒤에 normalization을 한다는 것이다. 이 순서만 바꿔서

mod, conv, norm, bias

로 한다면 bias가 생으로 들어간 뒤에 mod를 하고, normalization을 한 뒤에 bias를 한다. 즉, style(A)는 똑같은 연산을 거치는데 bias가 더 강력하게 되는 효과를 얻을 수 있다.
또한 mod에서 mean이 그다지 효과가 없어 없앴다고 한다. (in practice)
#### Instance normalization revisited
Instance Normalization은 채널별 정규화라는 것을 상기하자. 

두번째 개선책으로는 결국 위와같이 구조를 약간 변형하여도 주된 문제는 normalization이기 때문에 이를 개선한다. normalization이 문제라면 단순히 normalization을 없애면 해결이 될 테지만,
기존의 StyleGAN1에서의 강력한 특징중 하나가 스타일을 섞을수 있다는 것이다. 하지만 이는 AdaIN에서 이루어 지는 연산이므로 이를 포기하는 것은 말이 안됨.

따라서 instance정규화를 변형한다. 아래 오피셜 코드를 참고하면,
```python
# x가 feaature, y가 style. 
def modulated_conv2d_layer(x, y, fmaps, kernel, up=False, down=False, demodulate=True, resample_kernel=None, gain=1, use_wscale=True, lrmul=1,
                           fused_modconv=True, weight_var='weight', mod_weight_var='mod_weight', mod_bias_var='mod_bias'):
    assert not (up and down)
    assert kernel >= 1 and kernel % 2 == 1

    # Get weight.
    w = get_weight([kernel, kernel, x.shape[1].value, fmaps], gain=gain, use_wscale=use_wscale, lrmul=lrmul, weight_var=weight_var) # 단순히 He_init으로 conv weight초기화
    ww = w[np.newaxis] # [BkkIO] Introduce minibatch dimension.  # batch_size dimension 추가

    # Modulate.
    s = dense_layer(y, fmaps=x.shape[1].value, weight_var=mod_weight_var) # [BI] Transform incoming W to style. y가 style이다. style을 linear 한번 거친다. 
    s = apply_bias_act(s, bias_var=mod_bias_var) + 1 # [BI] Add bias (initially 1).  # bias apply.
    ww *= tf.cast(s[:, np.newaxis, np.newaxis, :, np.newaxis], w.dtype) # [BkkIO] Scale input feature maps.  # 기존의 conv weight에 scale을 곱해준다. 

    # Demodulate.
    if demodulate:
        d = tf.rsqrt(tf.reduce_sum(tf.square(ww), axis=[1,2,3]) + 1e-8) # [BO] Scaling factor.  # 논문의 demodulation 분모 부분
        ww *= d[:, np.newaxis, np.newaxis, np.newaxis, :] # [BkkIO] Scale output feature maps.

    # Reshape/scale input.
    if fused_modconv:
        x = tf.reshape(x, [1, -1, x.shape[2], x.shape[3]]) # Fused => reshape minibatch to convolution groups.
        w = tf.reshape(tf.transpose(ww, [1, 2, 3, 0, 4]), [ww.shape[1], ww.shape[2], ww.shape[3], -1])
    else:
        x *= tf.cast(s[:, :, np.newaxis, np.newaxis], x.dtype) # [BIhw] Not fused => scale input activations.

    # Convolution with optional up/downsampling.
    if up:
        x = upsample_conv_2d(x, tf.cast(w, x.dtype), data_format='NCHW', k=resample_kernel)
    elif down:
        x = conv_downsample_2d(x, tf.cast(w, x.dtype), data_format='NCHW', k=resample_kernel)
    else:
        x = tf.nn.conv2d(x, tf.cast(w, x.dtype), data_format='NCHW', strides=[1,1,1,1], padding='SAME')

    # Reshape/scale output.
    if fused_modconv:
        x = tf.reshape(x, [-1, fmaps, x.shape[2], x.shape[3]]) # Fused => reshape convolution groups back to minibatch.
    elif demodulate:
        x *= tf.cast(d[:, :, np.newaxis, np.newaxis], x.dtype) # [BOhw] Not fused => scale output activations.
    return x
```
주석처리로 설명한 것을 읽어보자. 

### Image quality and generator smoothness
PPL: StyleGAN1에서 나온 개념으로 StyleGAN1에서는 입력의 disentangle을 중요하게 여겻기 때문에 이를 측정하기 위해 입력이 조금만 달라질 경우 생성된 이미지의 차이를 구하는 측도.
작을 수록 좋다. 

#### Lazy regularization
subsection 이름 답게 직관적으로 regularization을 적게 한다고 보면 된다. 자세히 말하면, 우리가 학습을 할 때 주된 loss function이 있고, regularization 항을 붙여서 규제를 하는데
regularization의 computation cost를 줄이기 위해 논문에서는 미니배치 16번당 1번 regularization을 했다고 말한다. 성능에 영향은 당연히 없다고 한다. 

#### Path length reqularization
StyleGAN1에서 언급하였듯이 disentangle이 잘 될수록 더 좋은 이미지를 만들 수 있다. 이를 위해 StyleGAN2에서는 path length reqularization 항을 추가한다. 저자들은 $w \in \mathcal{W}$에서 
random한 방향으로 고정된 스텝만큼 움직였을때, $w$의 gradient deviation이 일정하게 움직인다면 가장 이상적으로 disentangle되어 있다고 주장한다. 이를 구하기 위해 우리가 처음에 배웠던 Jacobian 함수를 
활용한다. 
$\mathbb{J}_w = \delta g(w) / \delta w$
로 gradient 행렬을 표현 할 수 있고, 움직인 length가 방향에 상관없이 일정하다는 가정을 통하여 

$\mathcal{E}_{w, y \sim \mathcal{N}(0, I)} (\left\Vert\mathbb{J}^T_w y\right\Vert _2 - a)^2$

를 도출 할 수 있다. 이 path length regularization은 모델을 consistent하게 만드는 것도 있지만, 저자가 소개하는 장점은 모델 invert에 더 효과적이라고 말한다. (이는 section 5에서 다루는 내용.)
아래 그림을 보면 확연히 StyleGAN1보다 전체적으로 낮은 PPL score를 갖고 있음을 알 수 있다.

### Progressive growing revisited
StyleGAN1에서는 PG-GAN에서 사용한 growing 네트워크 구조를 활용하였다. 하지만 이 구조의 단점은 특정 위치에 대하여 위치정보를 강하게 갖고 있다는 것이다. (작은 부분들은 전체적인 면에서 변하기 힘듦.)
생각해보면, 작은 이미지는 전체적인 틀을 갖고 있으므로, 논문에서 보여주는 예시처럼, 사람의 구도가 바뀌는 동안, low resolution에서 이를 감지하고 생성하는 이미지 또한 전체적인 사람의
구도에 대한 변화를 나타낼 수 있다. 하지만 구도가 변하는 동안, 이빨과 같은 작은 부분은 거의 변하지 않는 것을 볼 수 있듯이, detail한 측면에서 localization이 심해져 이빨 같은 경우는 그냥 똑같은 이미지로 표현한다
(shift invariance). 이를 개선하기 위해 전체적인 네트워크 구조를 바꿈.  
  
#### Alternative network architecture
그 동안 생성 네트워크에서 뛰어난 성능을 보였던 구조 3가지를 소개한다.
1. Skip connections (U-Net)
2. Residual Network (resnet)
3. Hierarchical (StackGAN)
그 중에서 MSG-GAN을 모티브 삼아 변형된 네트워크 구조를 사용한다. 
![](/assets/images/2021-07-08-StyleGAN2/4.JPG)
주목할 점은 up, down sampling에서 bilinear interpolation을 썼다는 것. 
![](/assets/images/2021-07-08-StyleGAN2/5.JPG)
표의 결과를 정리하자면, 
1. Skip connection은 G에서 PPL score를 향상시킨다.
2. Residual은 D에서 FID score를 향상시킨다.
3. Residual은 G에서 성능을 하락시킨다.
즉, D: Residual, G: skip connection 쓰자. StyleGAN2도 이 구조를 따랐다.

### Projection of images to latent space
생성 네트워크의 역함수를 구하는 것은 굉장히 어려운 일이다. 역함수가 쉽게 구해지는 생성 네트워크가 잘 만들어진 네트워크라고도 한다. StyleGAN이 나온 이래로, 기존 연구들은 
StyleGAN을 역 projection 하는데에 중간 layer들의 w를 찾을 수 있다면 좋은 이미지를 생성한다고 말한다. 하지만 StyleGAN2는 mapping network가 생성한 $w$로 projection을 시도한다.

### Conclusion
이 section에서 주목할 것은 GAN도 적은 이미지로 생성하는 것이 다가왔다는 것이다. 수렴 안정성에 관한건 별로 중요하게 생각하지 않는듯.
